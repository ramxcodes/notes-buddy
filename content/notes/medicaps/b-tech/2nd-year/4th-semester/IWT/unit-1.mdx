## Introduction: Concept of WWW

The **World Wide Web (WWW)**, commonly referred to as the Web, is a system of interlinked hypertext documents and multimedia content that can be accessed via the Internet. It was invented by **Tim Berners-Lee** in 1989 while working at CERN.

### Key Features of the WWW
- **Hypertext System**: Links allow users to navigate between different web pages and resources.
- **Universal Accessibility**: Accessible globally through web browsers.
- **Multimedia Content**: Supports text, images, audio, videos, and interactive content.
- **Client-Server Model**: Web browsers act as clients, requesting resources hosted on servers.
- **Protocol-Based Communication**: Operates on the HyperText Transfer Protocol (HTTP/HTTPS).

### Components of the WWW
1. **Web Pages**: The basic building blocks of the Web, typically written in HTML (HyperText Markup Language).
2. **Web Servers**: Computers hosting websites and serving web pages upon request.
3. **Web Browser**: A software application used to access and display web pages (e.g., Google Chrome, Mozilla Firefox).
4. **URLs (Uniform Resource Locators)**: The address used to access a specific web page or resource.

### How the WWW Works
1. The user enters a **URL** into the web browser.
2. The browser sends an **HTTP/HTTPS request** to the web server hosting the resource.
3. The web server processes the request and sends back the requested content.
4. The browser renders the content for the user to view.

### Advantages of the WWW
- Easy access to vast amounts of information.
- Supports diverse multimedia content for enhanced user experience.
- Enables global connectivity and communication.
- Powers e-commerce, education, and social networking platforms.

### Limitations of the WWW
- Dependence on a stable Internet connection.
- Security vulnerabilities like hacking and data breaches.
- Digital divide due to unequal access to technology.

## HTTP Protocol: Request and Response

The **HyperText Transfer Protocol (HTTP)** is the foundation of communication on the World Wide Web. It defines the rules for transferring data between a client (browser) and a server over the Internet.

### Key Features of HTTP
- **Stateless Protocol**: Each request-response pair is independent, with no stored context.
- **Application Layer Protocol**: Operates at the top of the TCP/IP model.
- **Plain Text Communication**: Messages are typically in plain text, making them easy to read and debug.

### HTTP Request
An **HTTP Request** is a message sent by the client to the server to fetch or submit data.

#### Structure of an HTTP Request
1. **Request Line**:
   - Method: Specifies the action (e.g., `GET`, `POST`, `PUT`, `DELETE`).
   - URL: The resource location.
   - HTTP Version: The protocol version (e.g., `HTTP/1.1`).

2. **Headers**:
   - Provides additional information like the host, user agent, and content type.

3. **Body (Optional)**:
   - Contains data to be sent to the server (used in methods like `POST`).

#### Common HTTP Methods
- **GET**: Requests data from a server (read-only).
- **POST**: Submits data to the server (e.g., form submissions).
- **PUT**: Updates existing resources.
- **DELETE**: Removes specified resources.

### HTTP Response
An **HTTP Response** is a message sent by the server back to the client, providing the requested data or an error message.

#### Structure of an HTTP Response
1. **Status Line**:
   - HTTP Version: Protocol version (e.g., `HTTP/1.1`).
   - Status Code: Indicates the outcome (e.g., `200 OK`, `404 Not Found`).
   - Status Message: Describes the status.

2. **Headers**:
   - Contains metadata like content type, server name, and date.

3. **Body**:
   - Contains the actual data requested by the client (e.g., HTML, JSON, or other formats).

#### Common HTTP Status Codes
- **1xx (Informational)**: Request received, continuing process.
- **2xx (Success)**: Request was successful.
- **3xx (Redirection)**: Further action is needed to complete the request.
- **4xx (Client Error)**: The request contains errors.
- **5xx (Server Error)**: The server failed to process a valid request.

### How HTTP Request and Response Work
1. **Request Initiation**: The client sends an HTTP request to the server.
2. **Server Processing**: The server processes the request.
3. **Response Delivery**: The server sends an HTTP response back to the client.

## Web Browser Architecture

A **web browser** is a software application used to access, retrieve, and display content from the World Wide Web. It interprets HTML, CSS, JavaScript, and other web technologies to render web pages on the user’s device.

### Key Components of a Web Browser Architecture

1. **User Interface (UI)**:
   - The visible part of the browser, including the address bar, navigation buttons, bookmarks, and display area for the web page.
   - Handles user interactions and inputs.

2. **Browser Engine**:
   - Acts as a bridge between the user interface and the rendering engine.
   - Sends user requests (like navigating to a URL) to the rendering engine.

3. **Rendering Engine**:
   - Responsible for parsing HTML, CSS, and other web content to display the visual representation of the web page.
   - Examples of rendering engines:
     - Blink (used by Chrome and Edge)
     - WebKit (used by Safari)

4. **Networking Layer**:
   - Handles network communication, including making HTTP/HTTPS requests to fetch resources from the web server.
   - Manages caching and proxy settings.

5. **JavaScript Engine**:
   - Executes JavaScript code on web pages to enable interactivity.
   - Examples of JavaScript engines:
     - V8 (used by Chrome and Edge)
     - SpiderMonkey (used by Firefox)

6. **UI Backend**:
   - Draws basic widgets (like dropdowns and buttons) and integrates the UI with the rendering engine.
   - Often uses platform-specific libraries.

7. **Data Storage**:
   - Stores user data like cookies, bookmarks, cache, and browsing history.
   - Manages local storage and session storage for web applications.

8. **Security Module**:
   - Ensures secure browsing by implementing features like sandboxing, same-origin policy, and encryption for HTTPS.

### How a Web Browser Works
1. **User Input**: The user enters a URL in the address bar.
2. **Networking**: The browser makes an HTTP/HTTPS request to fetch the resource.
3. **Rendering**:
   - The rendering engine parses the HTML and CSS to build the **DOM (Document Object Model)** and **CSSOM (CSS Object Model)**.
   - Combines the DOM and CSSOM to create the **render tree**.
   - Paints the content on the screen using the render tree.
4. **JavaScript Execution**: The JavaScript engine executes any scripts on the page.
5. **User Interaction**: The browser updates the page dynamically based on user actions.

## Web Servers and Application Servers

Web servers and application servers are essential components in web-based systems. While they share some similarities, their roles and functionalities differ.

### Web Servers

A **web server** is software or hardware that serves web pages to clients (browsers) over the Internet or an intranet.

#### Key Features of a Web Server
- Handles HTTP/HTTPS requests from clients.
- Delivers static content such as HTML, CSS, JavaScript, and images.
- Supports dynamic content generation when paired with server-side scripts.

#### Common Web Servers
- **Apache HTTP Server**: Open-source, widely used for hosting websites.
- **Nginx**: High-performance server for serving static and dynamic content.
- **IIS (Internet Information Services)**: Developed by Microsoft, integrated with Windows OS.

#### Use Cases of Web Servers
- Hosting static websites.
- Serving files like documents and images.
- Acting as a reverse proxy for application servers.

---

### Application Servers

An **application server** is a software framework that provides an environment to run and manage applications. It handles business logic, application services, and database interactions.

#### Key Features of an Application Server
- Manages the execution of dynamic server-side applications.
- Supports various programming languages (e.g., Java, Python, PHP).
- Integrates with databases to fetch and process data.
- Provides services such as transaction management, messaging, and security.

#### Common Application Servers
- **Apache Tomcat**: Open-source, supports Java-based applications.
- **JBoss (WildFly)**: Java EE-compliant, supports enterprise-grade applications.
- **WebLogic**: Developed by Oracle for enterprise application deployment.

#### Use Cases of Application Servers
- Running dynamic web applications.
- Facilitating business logic processing.
- Managing API-based services for mobile or web applications.

---

### Differences Between Web Servers and Application Servers

| **Aspect**          | **Web Server**                                   | **Application Server**                                   |
|----------------------|-------------------------------------------------|---------------------------------------------------------|
| **Purpose**          | Serves static content and handles HTTP requests.| Executes dynamic applications and processes business logic.|
| **Content Delivery** | Static (e.g., HTML, CSS) and dynamic (limited). | Dynamic content generation and data processing.         |
| **Services**         | Basic HTTP handling, logging, and caching.      | Advanced services like transactions, messaging, and APIs.|
| **Examples**         | Apache HTTP Server, Nginx, IIS.                 | Apache Tomcat, JBoss (WildFly), WebLogic.               |

---

### Integration of Web Servers and Application Servers

In modern systems, web servers and application servers often work together:
1. The web server handles HTTP requests and serves static content.
2. Requests requiring business logic or database access are forwarded to the application server.
3. The application server processes the request and sends the response back to the web server, which delivers it to the client.

## Features of Web 2.0

**Web 2.0** represents the second generation of the World Wide Web, focusing on user-generated content, collaboration, and interaction. Unlike Web 1.0, which was primarily static, Web 2.0 is dynamic and user-centric.

### Key Features of Web 2.0

1. **User-Generated Content**:
   - Enables users to create, share, and interact with content.
   - Examples: Blogs, social media posts, and reviews.

2. **Rich User Experience (RUE)**:
   - Provides dynamic, interactive interfaces for better usability.
   - Utilises technologies like AJAX (Asynchronous JavaScript and XML) for seamless interactions without reloading pages.

3. **Collaboration and Sharing**:
   - Encourages community participation and collaboration.
   - Platforms like wikis and forums allow users to contribute and share knowledge.

4. **Social Networking**:
   - Facilitates connections between users through platforms like Facebook, LinkedIn, and Twitter.
   - Promotes the sharing of ideas, content, and discussions.

5. **Interactive Web Applications**:
   - Includes applications that function like desktop software but run on browsers (e.g., Google Docs, Trello).
   - Combines rich interfaces with powerful back-end functionality.

6. **Tagging and Folksonomy**:
   - Enables users to categorise and organise content using tags.
   - Examples: Hashtags on Twitter and Instagram.

7. **Syndication**:
   - Supports RSS (Really Simple Syndication) feeds for delivering updated content to users.
   - Allows users to subscribe to content updates.

8. **Openness and APIs**:
   - Offers open data sharing and interoperability through APIs (Application Programming Interfaces).
   - Examples include Google Maps API and Twitter API for integrating services.

9. **Scalability and Cloud Support**:
   - Designed for scalability to accommodate large volumes of users and content.
   - Leverages cloud computing for efficient resource management.

10. **Personalisation**:
    - Provides tailored content and recommendations based on user preferences and behaviour.
    - Examples: E-commerce platforms like Amazon and streaming services like Netflix.

---

### Advantages of Web 2.0
- Enhances user engagement and interactivity.
- Promotes knowledge sharing and collaboration.
- Allows for rapid content creation and dissemination.
- Drives innovation with open APIs and mashups.

### Limitations of Web 2.0
- Privacy concerns due to user-generated data.
- Over-dependence on user participation.
- Challenges in managing and verifying content authenticity.

## Internetworking with TCP/IP

**TCP/IP (Transmission Control Protocol/Internet Protocol)** is the fundamental suite of protocols that enable communication over the Internet and most modern computer networks. It is responsible for ensuring that data is transmitted between devices, routed correctly, and arrives intact.

### Key Components of TCP/IP

1. **Transmission Control Protocol (TCP)**:
   - **Reliability**: Ensures reliable communication by establishing a connection between sender and receiver.
   - **Segmentation and Reassembly**: Breaks large messages into smaller segments and reassembles them at the destination.
   - **Flow Control**: Manages the rate of data transmission to prevent network congestion.
   - **Error Detection**: Uses checksums to verify data integrity and correct transmission errors.

2. **Internet Protocol (IP)**:
   - **Routing**: Handles the addressing and routing of packets between networks.
   - **Addressing**: Assigns a unique IP address to each device for identification.
   - **Packetization**: Breaks data into smaller units (packets) for transmission across networks.

3. **Other Supporting Protocols**:
   - **User Datagram Protocol (UDP)**: Used for fast, connectionless communication with no guarantee of delivery (e.g., streaming).
   - **ICMP (Internet Control Message Protocol)**: Used for network diagnostics and error reporting (e.g., ping).
   - **ARP (Address Resolution Protocol)**: Resolves IP addresses to physical MAC addresses on a local network.
   - **DNS (Domain Name System)**: Translates human-readable domain names into IP addresses.

### Layers of the TCP/IP Model

1. **Application Layer**:
   - Deals with high-level protocols and provides services to end-users.
   - Examples: HTTP, FTP, SMTP, DNS.
   
2. **Transport Layer**:
   - Manages end-to-end communication and data flow between hosts.
   - TCP and UDP operate at this layer.

3. **Internet Layer**:
   - Handles logical addressing, routing, and packet forwarding.
   - IP operates at this layer, ensuring the correct delivery of packets.

4. **Network Interface Layer**:
   - Manages physical addressing, data link protocols, and the transmission of data on the network.
   - Involves Ethernet, Wi-Fi, and other local area network (LAN) technologies.

---

### How TCP/IP Works
1. **Data Transmission**: 
   - Data is divided into packets by the transport layer (TCP or UDP).
   - The packets are addressed by the internet layer (IP) and then transmitted over the network.

2. **Routing and Delivery**:
   - Routers use IP addresses to forward packets from the source to the destination.
   - The packets may pass through multiple intermediate devices to reach the destination.

3. **Reassembly**:
   - Upon arrival at the destination, the transport layer (TCP) reassembles the data into its original form.

4. **Error Handling**:
   - TCP ensures reliable data delivery by retransmitting lost or corrupted packets.

---

### Advantages of TCP/IP
- **Scalability**: Can support a large number of devices on the network.
- **Interoperability**: Works across different types of networks and devices.
- **Reliability**: Provides mechanisms for error detection and correction.
- **Flexibility**: Supports both connection-oriented (TCP) and connectionless (UDP) communication.

### Disadvantages of TCP/IP
- **Overhead**: TCP introduces additional overhead due to error-checking and retransmissions.
- **Complexity**: The TCP/IP stack can be complex, requiring proper configuration and management.

## Basics of DNS, SMTP, POP3

### DNS (Domain Name System)

The **Domain Name System (DNS)** is a system that translates human-readable domain names (e.g., www.example.com) into IP addresses (e.g., 192.168.1.1). It acts as the "phone book" of the internet, enabling users to access websites using easy-to-remember domain names instead of numeric IP addresses.

#### Key Features of DNS:
- **Name Resolution**: Resolves domain names to IP addresses.
- **Hierarchical Structure**: Organized in a tree structure with root, top-level domains (TLDs), and subdomains.
- **Caching**: Caches DNS queries to reduce lookup times and reduce network traffic.
- **Redundancy**: Multiple DNS servers ensure reliability and availability.

#### Types of DNS Records:
- **A Record (Address Record)**: Maps a domain name to an IPv4 address.
- **AAAA Record**: Maps a domain name to an IPv6 address.
- **MX Record (Mail Exchange)**: Specifies the mail server responsible for receiving email for a domain.
- **CNAME Record**: Alias record that maps a domain name to another domain name.

#### Example of DNS Resolution:
1. User types `www.example.com` into the browser.
2. The browser sends a DNS query to a DNS resolver.
3. The resolver queries DNS servers to find the corresponding IP address.
4. The IP address is returned, and the browser connects to the website.

---

### SMTP (Simple Mail Transfer Protocol)

**SMTP** is a protocol used for sending emails over the internet. It works by transferring email from the sender's email client to the recipient's email server.

#### Key Features of SMTP:
- **Mail Sending**: Primarily responsible for sending outgoing emails from client to mail server.
- **Server-to-Server Communication**: SMTP also enables mail servers to send emails to each other.
- **Text-based Protocol**: Operates in a simple text format, where commands and responses are exchanged between client and server.

#### How SMTP Works:
1. **Client to Server**: The email client connects to the SMTP server, providing the sender's address, recipient's address, and the message.
2. **Mail Delivery**: The server checks the recipient's domain (via DNS) to find the corresponding mail server.
3. **Server to Server**: If needed, the SMTP server communicates with other servers to deliver the email to the recipient's server.
4. **Email Storage**: The recipient's server stores the email in their mailbox for retrieval by the recipient.

#### Common SMTP Commands:
- **HELO**: Identifies the sender to the server.
- **MAIL FROM**: Specifies the sender’s email address.
- **RCPT TO**: Specifies the recipient’s email address.
- **DATA**: Sends the actual message content.

---

### POP3 (Post Office Protocol 3)

**POP3** is a protocol used by email clients to retrieve emails from a server. POP3 downloads emails from the server to the client and removes them from the server, making it ideal for offline access to emails.

#### Key Features of POP3:
- **Email Retrieval**: Downloads messages from the mail server to the client device.
- **Offline Access**: After emails are downloaded, they can be accessed offline, with no need for an active internet connection.
- **One-time Retrieval**: Once downloaded, emails are typically removed from the server (though some configurations may leave copies on the server).

#### How POP3 Works:
1. The email client connects to the POP3 server and authenticates the user.
2. The server sends the email messages to the client.
3. The client downloads the messages and stores them locally.
4. The server deletes the messages (unless configured to leave copies).

#### POP3 vs IMAP:
- **POP3**: Downloads emails and removes them from the server (best for offline use).
- **IMAP (Internet Message Access Protocol)**: Allows users to view and manage emails directly on the server without downloading them (ideal for multiple devices).

---

### Comparison of DNS, SMTP, and POP3

| **Protocol** | **Purpose**                               | **Direction**         | **Common Use**               |
|--------------|-------------------------------------------|-----------------------|------------------------------|
| **DNS**      | Resolves domain names to IP addresses     | Client → DNS Server    | Website and service access   |
| **SMTP**     | Sends email messages to a mail server     | Client → Mail Server   | Sending emails               |
| **POP3**     | Retrieves email from a server to a client | Server → Client        | Downloading emails           |
